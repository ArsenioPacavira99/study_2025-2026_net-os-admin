Целью данной работы является приобретение практических навыков по установке и конфигурированию DHCP-сервера, усвоение принципов автоматической выдачи сетевых настроек клиентам.

Выполнение работы

1. Подготовка лабораторного стенда

Лабораторный стенд был развёрнут на платформе Oracle VirtualBox с использованием Vagrant для автоматизации создания и настройки виртуальных машин. В качестве базовой операционной системы выбрана Fedora 39 (образ fedora/39-cloud-base).

1.1. Структура стенда

Стенд состоит из двух виртуальных машин:

Имя машины IP-адрес Роль
dhcp-server 192.168.56.10 DHCP-сервер (распределение сетевых настроек)
dhcp-client Динамический Клиентская машина для получения IP по DHCP

Обе машины подключены к внутренней виртуальной сети dhcp_lab с типом адаптера Host-Only. Сервер имеет статический IP-адрес, клиент получает настройки автоматически.

1.2. Конфигурация Vagrant

Файл Vagrantfile содержит следующую конфигурацию:

  • Провайдер: VirtualBox
  • Оперативная память: 512 МБ для каждой ВМ
  • Количество ядер CPU: 1
  • Сеть: для сервера настроен статический IP, для клиента - динамический
  • Синхронизация папок: каталог ./provision монтируется в /vagrant внутри ВМ

1.3. Предварительная настройка сервера

Перед установкой DHCP-сервера на машину dhcp-server были выполнены следующие действия:

  • Обновление пакетной базы: sudo dnf update -y
  • Установка сетевых утилит: sudo dnf install net-tools -y
  • Настройка статического IP-адреса через конфигурационный файл /etc/sysconfig/network-scripts/ifcfg-enp0s3
  • Отключение SELinux для упрощения отладки: sudo setenforce 0
  • Настройка файервола для разрешения DHCP-трафика: ```bash sudo firewall-cmd –add-service=dhcp –permanent sudo firewall-cmd –reload
